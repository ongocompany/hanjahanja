# WSD 합성 학습 데이터 v2 전략 (최종)

> v1은 동음이의어의 "소수 의미 불균형 해소"에 집중.
> v2는 **문맥 패턴 다양성**에 집중 — 축약어, 합성어, 지명, 직함, 띄어쓰기 변형 등.

---

## 1. v1에서 해결 안 된 문제 유형

### 1.1 축약어 (Abbreviation)
행정구역/기관명에서 뒷부분이 생략되면 다른 한자와 충돌.

| 축약 전 | 축약 후 | 올바른 한자 | 모델 예측 (오류) |
|---------|--------|-----------|---------------|
| 경기도 | 경기 | 京畿 | 競技 |
| 경기도지사 | 경기지사 | 京畿 | 競技 |
| 경기도당 | 경기+도당 | 京畿 | 競技 (형태소 분리 시) |
| 대전시 | 대전 | 大田 | 大戰? |
| 이천시 | 이천 | 利川 | 二千? |

### 1.2 직함/관직 합성어 (Title Compounds)
앞에 오는 단어가 뒤 직함과 결합할 때 의미가 확정됨.

| 합성어 | 올바른 | 모델 예측 (오류) |
|--------|--------|---------------|
| 수석대변인 | 首席 | 水石 |
| 수석비서관 | 首席 | 水石 |
| 수석연구원 | 首席 | 水石 |
| 수석졸업 | 首席 | 水石 |
| 사무국장 | 局長 | 國葬 |
| 김동연 지사 | 知事 | 支社 |
| 지사장 | 知事 | 支社? |

### 1.3 띄어쓰기 민감 오분류
붙여 쓰면 맞지만 띄어 쓰면 문맥이 끊겨서 오분류.
실제 뉴스 기사에는 띄어쓰기 오류가 빈번 → 모델이 대응해야 함.

| 붙여쓰기 (정답) | 띄어쓰기 (오류) | 올바른 한자 | 모델 예측 (오류) |
|---------------|---------------|-----------|---------------|
| 예비경선 → 競選 OK | 예비 경선 → 經線 NG | 競選 | 經線 |

### 1.4 고유어 한자 변환 금지 (블록리스트)
한자어 어원이 있지만 현대 한국어에서 절대 한자로 쓰지 않는 단어.

| 단어 | 잘못된 변환 | 조치 |
|------|-----------|------|
| 서울 | 暑鬱 | 블록리스트 추가 완료 (converter.ts) |

### 1.5 ETRI 오라벨 교정
ETRI가 잘못 라벨링한 데이터로 학습된 단어들.

| 단어 | 오라벨 | 실제 의미 | 오라벨 수 |
|------|--------|---------|---------|
| 의원 | 蟻援 (scode 05) | 議員 | 93개 |

## 2. v2 파이프라인 개요

```
[Step 1] LLM 문장 생성 (맞춤법 정확)
    ↓
[Step 2] 띄어쓰기 변형 후처리 (스크립트, 비율 통제)
    ↓
[Step 3] 교차 검증 (Round-Trip LLM 검증)
    ↓
[Step 4] 기존 train.jsonl 병합
    ↓
[Step 5] 재학습 → ONNX 변환 → 배포
```

핵심 원칙:
- **LLM은 깨끗한 문장만 생성** (맞춤법 정확)
- **띄어쓰기 노이즈는 후처리 스크립트가 주입** (비율 정밀 통제)
- **문체 다양성은 프롬프트에서 강제** (기사체 비중 높되 구어/SNS/공문서 혼합)

## 3. 생성 카테고리

### Category A: 지명 축약 패턴
**목표**: "경기" "대전" "이천" 등이 지명으로 쓰이는 문맥 학습

**대상 단어 수집 방법**:
1. 시/도/군/구 이름 중 한자어인 것 목록 추출
2. 해당 단어가 동음이의어 목록에 있는지 확인
3. 있으면 → 지명 문맥 문장 생성

**문장 패턴**:
- 단독 지명: "경기 남부에 폭우 예보가 내려졌다" (京畿)
- 지명+행정기관 복합: "경기도당 대의원대회가 열렸다" (京畿)
  → 형태소 분석기가 "경기+도당"으로 분리해도, 뒤 문맥(도당/도지사/도청)으로 학습
- 대조군: "경기에서 우승을 차지했다" (競技)

### Category B: 직함/관직/서열 합성어
**목표**: 직함/서열어가 직위와 결합할 때 올바른 한자 학습

**패턴**:
- [서열어] + [직함]: 수석 + (비서관/대변인/연구원) → 首席
- [기관] + [직위]: 사무 + 국장 → 局長, OO + 지사 → 知事
- 대조: 수석 + (정원/돌/계곡) → 水石

### Category C: ETRI 오라벨 역학습
**목표**: ETRI가 잘못 라벨링한 데이터의 올바른 버전 생성

**방법**:
1. 기존 학습 데이터에서 의심스러운 라벨 추출 (빈도 <=5인 한자로 라벨된 것)
2. 해당 문장들의 올바른 라벨로 재생성
3. 기존 오라벨 데이터 제거 + 올바른 데이터 추가

### Category D: 실전 발견 동음이의어 보강
**목표**: v1에서 커버했지만 여전히 실전에서 문제되는 단어 추가 보강

**대상**: 실제 뉴스 테스트에서 발견된 오분류 단어
- 경선 (競選/經線)
- 형이 테스트하면서 발견하는 것들 누적

## 4. 프롬프트 설계 (v2 최종)

### 4.1 공통 시스템 프롬프트

```
너는 한국어 자연어 처리(NLP) 모델 학습을 위한 '합성 데이터 생성 전문가'야.
너의 임무는 주어진 한자어 동음이의어가 포함된, 실생활에서 흔하게 쓰이는 자연스러운 문장을 생성하는 거야.

[핵심 규칙]
1. 각 의미별로 문맥만 읽어도 어떤 뜻인지 명확히 구별 가능해야 함
2. 문체 배분 (아래 비율을 반드시 지켜):
   - 뉴스 기사체 40% (보도문, 속보, 사설)
   - 문어체 20% (보고서, 논문, 교과서)
   - 구어체 20% (일상 대화, 인터뷰)
   - SNS/블로그체 10% (댓글, 트윗, 블로그)
   - 공문서체 10% (법률, 관보, 공지)
3. 타겟 단어가 반드시 문장에 자연스럽게 포함되어야 함
4. 같은 패턴 반복 금지 — 다양한 주어, 서술어, 상황 사용
5. 각 문장은 15자 이상 100자 이하
6. 맞춤법과 띄어쓰기는 정확하게 작성 (노이즈는 후처리로 추가함)
7. 출력은 반드시 JSON 배열만 출력. 다른 설명, 인사말, 마크다운 절대 금지
8. JSON 키: "sentence" (한글 문장), "word" (타겟 단어), "sense" (의미 번호: A 또는 B)
```

### 4.2 Category A 전용: 지명 축약

```
[추가 규칙 — 지명 패턴]
1. 지명 문장의 50%는 "도/시/군/구" 없이 지명임이 명확한 문맥
   - 예: "경기 남부", "경기도청", "경기 지역 주민"
2. 지명 문장의 30%는 지명+행정기관 복합어 포함
   - 예: "경기도당", "경기도지사", "경기도의회", "경기도청"
3. 나머지 20%는 자유 문맥

단어: "{word}"
의미 A (지명): {hanja_A} — "{definition_A}"
의미 B (일반): {hanja_B} — "{definition_B}"

의미 A {N_a}개, 의미 B {N_b}개 생성해줘.
```

### 4.3 Category B 전용: 직함/서열 합성어

```
[추가 규칙 — 직함 패턴]
1. 직함 문장: 반드시 뒤에 직위/역할/기관이 오는 패턴 포함
   - 예: "수석 비서관", "사무국장", "OO 지사"
2. 직함과 결합하는 단어를 최대한 다양하게:
   - 비서관, 대변인, 연구원, 보좌관, 검사, 판사, 부장, 과장, 팀장 등
3. 인명 + 직함 패턴도 포함:
   - 예: "김철수 지사", "박영희 국장"
4. 일반 문장: 원래 한자 의미 그대로
   - 예: "계곡의 수석이 아름답다" (水石)

단어: "{word}"
의미 A (직함/서열): {hanja_A} — "{definition_A}"
의미 B (일반): {hanja_B} — "{definition_B}"

의미 A {N_a}개, 의미 B {N_b}개 생성해줘.
```

### 4.4 Category D 전용: 일반 동음이의어

v1 프롬프트와 동일 (docs/reference/synthetic-data-prompt-design.md 참조)

## 5. 후처리: 띄어쓰기 변형 주입

### 5.1 왜 후처리인가?
- LLM에게 "일부만 띄어쓰기 틀리게" 요청 → 비율 통제 불가 (배치 호출마다 랜덤)
- 후처리 스크립트 → 정확히 원하는 비율로 변형 가능

### 5.2 변형 규칙
생성된 문장 중 일정 비율(예: 10~15%)을 선택하여 합성어 경계에서 띄어쓰기 변형:

```python
# 예시 변형 규칙
SPLIT_RULES = {
    # 합성어 분리 (붙여쓰기 → 띄어쓰기)
    '예비경선': '예비 경선',
    '사무국장': '사무 국장',
    '수석대변인': '수석 대변인',
    '경기도당': '경기 도당',
    '경기도지사': '경기 도지사',
    # ... 타겟 단어별 분리 패턴 정의
}

# 변형 적용 (라벨은 동일하게 유지)
for sample in generated_data:
    if random() < 0.12:  # 12% 확률로 변형
        for original, spaced in SPLIT_RULES.items():
            sample['sentence'] = sample['sentence'].replace(original, spaced)
```

### 5.3 핵심 원칙
- **라벨은 변경하지 않음**: 띄어쓰기만 바뀌고 의미는 동일
- **합성어 경계에서만 분리**: 무작위 위치가 아님
- **비율 정밀 통제**: 10~15% 정도가 적당 (실제 뉴스 띄어쓰기 오류율 참고)

## 6. 대상 단어 목록

### 6.1 실전 테스트 발견 (확정)
| 단어 | 의미 충돌 | 카테고리 | 조치 |
|------|---------|---------|------|
| 경기 | 京畿/競技 | A (지명) | 합성 데이터 |
| 국장 | 局長/國葬 | B (직함) | 합성 데이터 |
| 경선 | 競選/經線 | D (일반) + 띄어쓰기 후처리 | 합성 데이터 |
| 지사 | 知事/支社 | B (직함) | 합성 데이터 |
| 서울 | 暑鬱 | - | 블록리스트 완료 |

### 6.2 재확인 필요 (형 테스트 중)
- 수석 (首席/水石) — v1에서 이미 보강했을 수 있음
- 의원 (議員/醫院) — v1 이후 잘 되는 것 같음

### 6.3 자동 추출 대상
1. **지명 축약**: 시/도/군 이름 intersect 동음이의어 목록
2. **직함 합성**: 서열/직위 관련 한자어 intersect 동음이의어 목록
3. **ETRI 오라벨 후보**: 빈도 <=5 한자로 예측되는 학습 데이터
